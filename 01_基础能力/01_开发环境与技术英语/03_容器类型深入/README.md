# 容器类型深入

> **所属路径**：`01_基础能力/01_开发环境与技术英语/03_容器类型深入`
> **包含知识点**：5 个

---

## 概述

在 [编程语言基础](../01_编程语言基础/README.md) 中，我们学会了使用 Python 的内置容器——列表（list）、字典（dict）、元组（tuple）和集合（set）。这四种基础容器已经能覆盖大部分日常编程场景。但当你开始处理更复杂的数据结构需求时，可能会发现自己在"重复造轮子"：需要一个有默认值的字典？需要统计元素出现次数？需要一个先进先出的队列？需要一个不可变的记录类型？

好消息是，Python 标准库中的 `collections` 模块以及 `dataclasses` 等工具，早已为这些需求提供了优雅的解决方案。本主题将带你从基础容器"升级"到这些更强大、更专业的容器工具，同时学习如何通过自定义容器来满足特殊场景的需求。

本主题还将帮助你建立对容器性能的直觉——不同容器在增删查改时的效率差异巨大，选对容器往往比优化算法更有效。

## 学习建议

- **学习顺序**：建议按照 01 → 05 的顺序学习。01 collections 模块是本主题的核心，提供了最实用的高级容器；02 数据类与具名元组是定义结构化数据的利器，在实际项目中使用频率极高；03 枚举类型相对独立且简单，适合穿插学习；04 自定义容器属于进阶内容，需要在理解前面的容器后再学习；05 容器性能对比是总结性质的内容，帮助你建立全局视角。
- **时间分配**：每个知识点预计 40–60 分钟（含动手实践），整个主题约需 4–5 小时。
- **学习方法**：容器类型的学习关键是"用起来"——每学一个新容器，就想想自己过去写过的代码中，有哪些地方可以用它来简化。建议在 Python 交互式环境中逐个尝试，并与内置容器对比使用。
- **常见困难点**：
  - `defaultdict` 的工厂函数参数容易理解错——它接收的是一个**可调用对象**（如 `int`、`list`），而不是一个默认值。
  - `dataclass` 和 `NamedTuple` 功能相似但设计哲学不同——前者是可变的类，后者是不可变的元组，选择时需要考虑是否需要修改属性。
  - 自定义容器需要理解 Python 的 **数据模型（Data Model）** ，特别是 `__getitem__`、`__len__` 等魔术方法。
