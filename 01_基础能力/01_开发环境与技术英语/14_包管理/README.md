## 概述

Python 之所以能在数据科学、Web 开发和人工智能等领域大放异彩，很大程度上得益于其庞大而活跃的第三方包生态。然而，"安装一个包"远没有看上去那么简单——你需要知道如何选择正确的版本、如何避免依赖冲突、如何让团队中的每个人都能复现相同的环境，以及如何在网络受限的环境下高效获取包。这些正是**包管理**要解决的核心问题。

本主题紧承 [虚拟环境](../13_虚拟环境/) 的内容。在虚拟环境中，我们学会了为项目创建隔离的 Python 环境；而在包管理中，我们将学习如何在这些环境中精确地安装、约束、记录和分发依赖。掌握包管理是 Python 开发者的必备基本功，也是后续 [版本控制](../15_版本控制/) 中将项目配置纳入 Git 管理的前提。

## 学习建议

建议按照知识点编号顺序学习，预计总学习时间约 2.5 小时：

1. **安装与卸载**（30 分钟）：这是包管理的起点，务必动手在虚拟环境中实际操作 `pip install` 和 `pip uninstall`，体会全局安装与虚拟环境安装的区别。
2. **版本约束**（35 分钟）：理解语义版本号是后续内容的基础。重点关注 `~=` 兼容版本运算符和版本冲突的诊断方法。
3. **依赖清单与锁定文件**（40 分钟）：这是团队协作中最关键的知识点。务必理解"抽象依赖"与"精确快照"的区别，并亲手使用 `pip-compile` 生成锁定文件。
4. **私有源与镜像配置**（35 分钟）：国内开发者必学内容。建议实际配置一次镜像源，验证加速效果。

**常见困难点**：版本冲突的诊断往往让初学者感到困惑。遇到安装失败时，不要急着搜索错误信息，先尝试阅读 pip 输出的冲突报告，理解哪两个包的版本要求产生了矛盾。
