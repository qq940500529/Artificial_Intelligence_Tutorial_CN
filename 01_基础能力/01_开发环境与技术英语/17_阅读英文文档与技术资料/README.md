## 概述

当你开始编写稍具规模的 Python 程序时，会越来越频繁地面对同一个问题：**遇到新库、新 API、新概念时该去哪里学？** 答案并不是打开搜索引擎输入中文关键词——那样你往往会被大量过时博客、错误拼凑的代码片段包围；真正可靠的来源是项目的**官方英文文档**、**源码中的 docstring**、**issue 讨论**，以及最前沿的**英文技术博客与论文**。这些材料的共同特点是：信息密度高、模式化程度高、但对英语不熟悉的读者"望而生畏"。

本主题承接 [阅读文档（00/02_英语基础）](../../../00_高中复习/02_英语基础/03_阅读文档/) 对通用阅读框架的启蒙，把它升级为一套面向专业 Python/AI 工程师的实战阅读策略。我们不追求把每个单词都查清楚，而是训练四项核心技能：**快速定位**（在海量页面中 30 秒找到目标）、**接口精读**（看懂函数签名的每一个细节）、**文章摘要**（10 分钟读完一篇技术博文并提取要点）、**论文阅读**（用结构化方法读懂 arXiv 预印本）。这些技能是你进入 [论文阅读（04 阶段）](../../../04_持续研究/01_研究与持续学习/01_论文阅读/) 和 [源码阅读（04 阶段）](../../../04_持续研究/01_研究与持续学习/06_源码阅读/) 的专业基础。

## 教学方法说明

本主题延续 **ESP（专门用途英语）** 教学方法论，同时强化面向专业场景的阅读策略：

| 策略 | 说明 | 在课程中的体现 |
| ---- | ---- | -------------- |
| **结构优先（Structure First）** | 先掌握文本骨架，再填充细节 | 每课先给出对象的典型结构图，再讲每一块 |
| **语块记忆（Lexical Chunking）** | 把固定表达作为整体记忆 | 每课列出"高频语块"对照表 |
| **词根解码（Morphological Decoding）** | 用词根词缀推理陌生术语 | 第 02/04 课提供词根解码表 |
| **分层阅读（Layered Reading）** | 先扫读、再略读、最后精读 | 第 03/04 课用"三遍法"读博文与论文 |
| **输出驱动（Output-Driven）** | 通过摘要、笔记巩固输入 | 第 03 课包含"三句话摘要"任务 |
| **间隔复习（Spaced Repetition）** | 按遗忘曲线安排复习 | 每课末附间隔复习建议表 |

## 学习建议

**推荐学习顺序**：严格按 01 → 02 → 03 → 04 顺序学习。快速定位是一切文档阅读的入口；接口精读是 Python 开发最高频的阅读场景；技术文章摘要把阅读从"查资料"扩展到"跟前沿"；论文阅读把视野进一步延伸到学术一线。难度逐级上升。

**时间分配**：四个知识点合计约 4–5 小时。建议分两天完成——第一天学 01 和 02（工具/接口类高频场景），第二天学 03 和 04（叙事/论证类深度阅读）。

**常见困难**：

- **想逐字读完**：英文技术资料的信息密度很不均匀，80% 的价值集中在 20% 的句子上。本主题反复训练的就是"跳过不重要部分"的勇气。
- **被专业术语吓到**：绝大多数 AI 术语都可以用词根词缀拆解（例如 `autoencoder = auto- + encode + -er`），不必全背。
- **读完即忘**：在 [总结与记笔记](../../../00_高中复习/02_英语基础/04_总结与记笔记/) 中学过的术语卡片与章节摘要方法，在本主题会被反复使用。
